  \begin{block}{Recursividad}
    Según Wikipedia:
    \begin{quote}
      ``\textbf{Recursión} o \textbf{Recursividad} es la
      forma en la cual se especifica un proceso basado
      en su propia definición. La \textbf{recursión} tiene esta característica discernible en términos de autorreferencialidad, autopoiesis, fractalidad, o en otras palabras, construcción
      a partir de un mismo tipo.''
    \end{quote}
  \end{block}
\end{frame}

\begin{frame}
  \frametitle{Unidad 5. Recursividad}
  \framesubtitle{Recursividad}

  \begin{exampleblock}{factorial}
    Según Wikipedia:
    \begin{quote}
      Se desea calcular $n!$ (el factorial de $n$, se define como
      el producto de todos los enteros positivos de $1$ a $n$).
      Se puede definir el problema de forma recurrente como $n(n-1)!$;
      como $(n-1)!$ es menor que $n$ podemos aplicar inducción por
      lo que disponemos del resultado. El caso base es $0!$ es $1$.
    \end{quote}
  \end{exampleblock}
\end{frame}

\begin{frame}[fragile]
  \frametitle{Unidad 5. Recursividad}

  \begin{exampleblock}{Implementación de factorial}
    La siguiente es la implementación de la función factorial
    es scala.
\begin{lstlisting}[language=scala]
def factorial(v: Int):Int = v match {
case 0 => 1
case 1 => 1
case n => n * factorial(n-1)
}
\end{lstlisting}
  \end{exampleblock}
  
\end{frame}

  \begin{enumerate}
  \item<1-> Una función recursiva tiene siempre un caso base. ¿Cuál es el caso base?
  \item<2-> ¿Cuál es el paso recursivo?
  \item<3-> ¿Es necesario tener dos casos base?
  \item<4-> ¿Si eliminamos el caso base con $1$ el código resultante sería más efectivo?
    
  \item<5-> ¿Si dejamos el caso base $1$ y eliminamos el caso base $0$ el anterior programa esta correcto?
  \end{enumerate}

  \begin{exampleblock}{Máximo común divisor}
    Se puede utilizar la definición de algoritmo de Euclides, que tiene
    varias definiciones.
    \[
      \text{mcd}(a,b) = \begin{cases}
        a & \text{Si} b = 0 \\
        \text{mcd}(b,a\ \text{mod}\ b) & \text{De otra forma} \\
      \end{cases}
    \]
    Implemente el siguiente código:
\begin{lstlisting}[language=scala]
def mcd(a:Int, b:Int):Int = ???
\end{lstlisting}
  \end{exampleblock}
\end{frame}

\begin{frame}[fragile]
  \frametitle{Unidad 5. Recursividad}
  \framesubtitle{Recursividad}
  \begin{exampleblock}{Implementación de \texttt{mcd}}
\begin{lstlisting}[language=scala]
def mcd(a:Int, b:Int):Int = b match {
case 0 => a
case _ => mcd(b, a % b)
}
\end{lstlisting}
  \end{exampleblock}
\end{frame}

  \begin{exampleblock}{Potencia de un número}
    Una definición interesante el cálculo de la potencia $x^y$ de
    forma recursiva:
\begin{lstlisting}[language=scala]
def potAux(x:Int, t:Int, a:Int):Int = t match {
  case 0 => 1
  case 1 => a
  case _ => potAux(x,t-1,a*x)
}

def pot(x:Int, y:Int):Int = {  
  potAux(x,y,x)
}
\end{lstlisting}
  \end{exampleblock}
\end{frame}

  \item<2-> Muchos tipos de datos se pueden definir de forma recursiva.
  \item<3-> Por ejemplo: una lista de valores enteros.

\begin{frame}
  \frametitle{Unidad 5. Recursividad}
  \framesubtitle{Recursividad - Tipos de datos recursivos}

  \begin{block}{Definición de una lista de enteros}
    \[
      ListaEnt = \begin{cases}
        Vacia & \\
        Entero :: listaEnt & \\
      \end{cases}
    \]
  \end{block}
\end{frame}

\begin{frame}[fragile]
  \frametitle{Unidad 5. Recursividad}
  \framesubtitle{Recursividad - Tipos de datos recursivos}
  
  \begin{block}<1->{Definición de una lista de enteros - Scala}
\begin{lstlisting}[language=scala]
trait ListaEnt
case object Vacia extends ListaEnt
case class ConsEnt(elem:Int, lista:ListaEnt) extends ListaEnt
\end{lstlisting}
    \vspace{-3ex}
    \begin{enumerate}
    \item<2-> La línea $1$ se define un tipo de dato:
      \texttt{ListaEnt}. ¿Qué se define la línea $2$?
    \item<3-> La línea $3$: ¿Qué sé está definiendo?
    \item<4-> Si el tipo que se acaba de definir es un tipo recursivo: ¿Cuál es la parte
      base y cuál es la recursiva?
    \end{enumerate}
  \end{block}
\end{frame}

\begin{frame}[fragile]
  \frametitle{Unidad 5. Recursividad}
  \framesubtitle{Recursividad - Tipos de datos recursivos}

  \begin{exampleblock}{Implementación de \texttt{longitudListaEnt}}
\begin{lstlisting}<1->[language=scala]
def longitudListaEnt(lst:ListaEnt):Int = lst match {
   case Vacia => 0
   case ConsEnt(e,lstp) => 1 + longitudListaEnt(lstp)
}
\end{lstlisting}
    \vspace{-3ex}
    \begin{enumerate}
    \item<2-> En la línea 2: ¿\texttt{Vacia} es un valor, una variable o un comodín?
    \item<3-> En la línea 3: ¿Qué significado tiene \texttt{ConsEnt}?
    \item<4-> En la línea 3: ¿Qué tipo tiene la variable \texttt{e}?
    \item<5-> En la línea 3: ¿Qué tipo tiene la variable \texttt{lstp}?
    \item<6-> ¿Cuál es el caso base y cuál es el caso recursivo?
    \end{enumerate}
  \end{exampleblock}
\end{frame}

  \begin{block}<1->{Definición de un árbol binario}
    La siguiente es la definición de un árbol binario de enteros.
\begin{lstlisting}[language=scala]
trait ArbolEnt
case object ArbolVacio extends ArbolEnt
case class  HojaEnt(e:Int) extends ArbolEnt
case class  Bin(i:ArbolEnt,d:ArbolEnt) extends ArbolEnt
\end{lstlisting}
    \begin{enumerate}
    \item<2-> ¿Cuál es la parte base de \texttt{ArbolEnt}?
    \item<3-> ¿Cuál es la parte recursiva de \texttt{ArbolEnt}?
    \item<4-> ¿Se permiten árboles vacíos?
    \item<5-> ¿Se permiten árboles con un solo valor entero?
    \item<6-> ¿Cómo sería un árbol que guarde 3 elementos enteros cómo: $1,2,3$?
    \end{enumerate}
  \end{block}
\end{frame}

\begin{frame}[fragile]
  \frametitle{Unidad 5. Recursividad}
  \framesubtitle{Recursividad - Tipos de datos recursivos}

  \begin{exampleblock}<1->{Función que calcula la altura de un árbol de enteros \texttt{ArbolEnt}}
\begin{lstlisting}[language=scala]
def alturaArbolEnt(a:ArbolEnt):Int = a match {
  case ArbolVacio => 0
  case HojaEnt(_) => 1
  case Bin(i,d)   => max(alturaArbolEnt(i), alturaArbolEnt(d)) + 1
}
\end{lstlisting}
    \begin{enumerate}
    \item<2-> ¿Cuáles son los casos bases?
    \item<3-> ¿Por qué en el caso de \texttt{HojaEnt} no interesa el valor interno?
    \item<4-> ¿Por qué en el caso de \texttt{Bin} si interesa ambos valores?
    \item<5-> ¿Por qué se debe obtener el máximo de ambos subárboles?
    \end{enumerate}
  \end{exampleblock}
\end{frame}
